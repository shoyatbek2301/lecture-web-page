How to Play Space Dodger

Space Dodger is a simple yet challenging arcade-style game where you control a spaceship and must avoid incoming asteroids. The longer you survive, the higher your score!

Controls

Device Control Action
Keyboard Left Arrow Key Move spaceship left
Keyboard Right Arrow Key Move spaceship right
Mobile/Tablet Swipe Left Move spaceship left
Mobile/Tablet Swipe Right Move spaceship right

Game Rules

  • Your spaceship (green circle) appears at the bottom of the screen
  • Asteroids (red circles) fall from the top of the screen at random positions
  • Each asteroid you successfully avoid adds 10 points to your score
  • If an asteroid hits your spaceship, the game is over
  • As your score increases, the game becomes progressively harder:
    • Asteroids fall faster
    • Asteroids appear more frequently

Tips

  • Stay alert and plan your movements in advance
  • Look for patterns in the falling asteroids
  • Try to position yourself in areas with fewer incoming asteroids
  • Be prepared for the increasing difficulty as your score grows
  • On mobile devices, use small, controlled swipes for precise movement

Game Development Report

Project Overview

Space Dodger is a browser-based arcade game developed as a web programming assignment. The game demonstrates core principles of interactive web development, including HTML structure, CSS styling, and JavaScript functionality.

Technical Implementation

The game is built using standard web technologies:

  • HTML5: Provides the structural foundation for the game elements
  • CSS3: Handles all visual styling and animations
  • JavaScript: Powers the game logic, controls, collision detection, and scoring system

No external libraries or frameworks were used in the development of this game, making it lightweight and fast-loading.

Key Features

  • Responsive Design: The game adapts to different screen sizes and devices
  • Touch Controls: Support for both keyboard and touch-based input methods
  • Dynamic Difficulty: Game speed and asteroid frequency increase as the player's score rises
  • Score Tracking: Real-time score display and final score on game over
  • UI Components: Start screen, game over screen, and instructional elements

Development Challenges

Several challenges were addressed during the development process:

  • Collision Detection: Implementing efficient collision detection between the player and asteroids
  • Cross-device Compatibility: Ensuring consistent gameplay experience across desktop and mobile devices
  • Performance Optimization: Managing the game loop and object creation/removal for smooth gameplay
  • Progressive Difficulty: Creating a balanced difficulty curve that challenges players without feeling unfair

Future Enhancements

Several features could be added to enhance the game in future updates:

  • Power-ups and special abilities
  • Multiple levels with different themes
  • High score leaderboard using local storage
  • Sound effects and background music
  • Visual enhancements with sprite-based graphics
  • Additional obstacle types beyond basic asteroids

Learning Outcomes

This project demonstrated practical application of:

  • Game loop implementation using requestAnimationFrame
  • DOM manipulation for creating and managing game elements
  • Event handling for user input across different devices
  • State management for game progression
  • Responsive design principles for cross-device compatibility
